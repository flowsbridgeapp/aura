# 🔐 Безопасность Supabase в Aura P2P Messenger

## ✅ Выполненные улучшения

### 1. Вынос секретных ключей в .env
- **Файл `.env`**: содержит реальные ключи (добавлен в `.gitignore`)
- **Файл `.env.example`**: шаблон для разработчиков без чувствительных данных
- **Код**: использует `import.meta.env.VITE_*` вместо хардкода

### 2. Префикс VITE_
Все переменные используют префикс `VITE_`, что позволяет Vite безопасно внедрять их в клиентский код.

## 📋 Инструкция для разработчиков

### Первый запуск:
```bash
# 1. Скопируйте пример
cp .env.example .env

# 2. Отредактируйте .env и укажите ваши ключи Supabase
# VITE_SUPABASE_URL=ваш_url
# VITE_SUPABASE_KEY=ваш_anon_key

# 3. Запустите проект
npm run dev
```

### Для новых разработчиков в команде:
1. Получите `.env` от ведущего разработчика **безопасным способом** (не через Git!)
2. Или настройте свой собственный проект Supabase
3. Никогда не коммитьте `.env` в репозиторий

## 🔒 Рекомендации по безопасности

### 1. Используйте RLS (Row Level Security)
Включите RLS в вашей базе данных Supabase:
```sql
ALTER TABLE messages ENABLE ROW LEVEL SECURITY;
```

### 2. Создайте политики доступа
Пример политики для таблицы сообщений:
```sql
CREATE POLICY "Пользователи могут читать сообщения своей комнаты"
ON messages FOR SELECT
USING (room_id = current_setting('app.current_room'));
```

### 3. Используйте только anon key на клиенте
- ✅ `sb_publishable_*` (anon key) — можно использовать в браузере
- ❌ `service_role` ключи — **никогда** не используйте на клиенте!

### 4. Ограничьте доступ по домену
В настройках Supabase Project Settings → API:
- Добавьте разрешённые домены (Production, Staging)
- Включите "Enforce secure cookies" для production

## 🚨 Что делать при утечке ключей

1. **Немедленно** перевыпустите ключи в Dashboard Supabase:
   - Project Settings → API → Reset keys
   
2. Обновите `.env` файл во всех средах

3. Проверьте логи на предмет подозрительной активности

## 🛡️ Дополнительные меры

### Для production:
- Настройте CORS в Supabase
- Используйте Cloudflare или аналогичный CDN
- Включите rate limiting
- Мониторьте использование API в Dashboard

### Аудит безопасности:
- Регулярно проверяйте политики RLS
- Анализируйте логи запросов
- Обновляйте зависимости (`npm audit`)
